# sql-library-management-system
A library management database project built using SQL with multiple tables, relationships, and analytical queries.
# sql-library-management-system Project

## Project Overview
This project demonstrates a simple Library Management Database using SQL.
The database stores information about branches, employees, books, customers, book issues, and returns.

## Database Tables
The database contains the following tables:
* Branch
* Employee
* Books
* Customer
* IssueStatus
* ReturnStatus

## Features
* Track book availability
* Manage customer registrations
* Record issued and returned books
* Analyze library data using SQL queries

## SQL Concepts Used
* Primary Keys
* Foreign Keys
* Joins
* Group By
* Having
* Subqueries
* Sorting and Filtering
* Pattern matching using LIKE

## Sample Queries
Some example queries included in this project:
* Retrieve available books
* List employees by salary
* Find books issued by customers
* Count books by category
* Customers who registered before 2022
* Books issued in June 2023
* Books containing "History" in the title

## Tools Used
* MySQL

## Conclusion
This project demonstrates the design and implementation of a Library Management System using SQL. It highlights the use of relational database concepts such as table relationships, joins, grouping, and data filtering to efficiently manage and analyze library data.
